# 📖 Introduction

JavaScript is one of the most widely used programming languages in the world. Understanding its history helps explain how it evolved from a simple scripting language into a powerful language used for web, server-side, mobile, desktop, cloud, and AI applications.

This chapter explores the origin of JavaScript, the people behind its development, major milestones, and the evolution of the ECMAScript standard.

# The Early Web

In the early 1990s, websites were mostly **static**.

A webpage could display:

- Text
- Images
- Hyperlinks

However, users could not interact with pages without sending requests to the server. Even simple actions like validating a form required a page reload.

Developers needed a language that could run directly in the browser to make websites more interactive.

---

# The Birth of JavaScript

JavaScript was created in **1995** by **Brendan Eich** while working at **Netscape Communications Corporation**.

Remarkably, the first version of JavaScript was developed in just **10 days**.

Its original goal was to provide a lightweight scripting language that could run inside web browsers and make web pages interactive.

---

# Original Name

JavaScript was not originally called JavaScript.

Its names changed several times:

| Version | Name |
|----------|------|
| Initial Development | Mocha |
| Beta Version | LiveScript |
| Official Release | JavaScript |

The name **JavaScript** was chosen as part of a marketing strategy because the Java programming language was becoming very popular at that time.

Despite the similar names, **Java and JavaScript are completely different languages**.

---

# Netscape Navigator

JavaScript was first introduced in the **Netscape Navigator 2.0** web browser.

It allowed developers to:

- Validate forms
- Display pop-up messages
- Respond to user actions
- Manipulate web pages
- Create animations

This transformed static websites into interactive web applications.

---

# Browser Competition

After JavaScript's success, Microsoft introduced its own implementation called **JScript** for **Internet Explorer**.

Different browsers implemented JavaScript differently, causing compatibility issues for developers.

This period became known as the **Browser Wars**.

---

# Standardization

To solve browser compatibility problems, JavaScript was standardized.

In **1997**, the language specification was submitted to **ECMA International**.

The standardized version became known as **ECMAScript (ES)**.

---

# What is ECMAScript?

**ECMAScript** is the official standard that defines how JavaScript should work.

JavaScript is an implementation of the ECMAScript specification.

Think of it this way:

```text
ECMAScript
      │
      ├── JavaScript
      ├── JScript (historical)
      └── Other implementations
```

---

# Evolution of ECMAScript

## ECMAScript 1 (1997)

- First standardized version
- Basic language features

---

## ECMAScript 2 (1998)

- Editorial improvements
- Better international support

---

## ECMAScript 3 (1999)

Major improvements included:

- Regular Expressions
- Exception Handling
- Better String methods
- Better Arrays

This version remained the industry standard for many years.

---

## ECMAScript 4

This version was planned but **never officially released** due to disagreements about its design.

Many proposed features were later introduced gradually in future versions.

---

## ECMAScript 5 (2009)

A major update that introduced:

- Strict Mode
- JSON support
- Array iteration methods
- Object methods
- Property descriptors

Example:

```javascript
"use strict";
```

---

## ECMAScript 6 (ES2015)

This was the biggest update in JavaScript history.

New features included:

- let
- const
- Arrow Functions
- Classes
- Modules
- Template Literals
- Promises
- Default Parameters
- Destructuring
- Spread Operator
- Rest Parameters

Example:

```javascript
const greet = (name) => `Hello ${name}`;
```

---

# Modern JavaScript

After ES2015, JavaScript adopted **annual releases**.

Examples:

| Version | Year |
|----------|------|
| ES2016 | 2016 |
| ES2017 | 2017 |
| ES2018 | 2018 |
| ES2019 | 2019 |
| ES2020 | 2020 |
| ES2021 | 2021 |
| ES2022 | 2022 |
| ES2023 | 2023 |
| ES2024 | 2024 |
| ES2025 | 2025 |

Each version introduced improvements and new language features while maintaining backward compatibility.

# JavaScript Today

JavaScript is no longer limited to web browsers.

Today it is used for:

- Frontend Development
- Backend Development (Node.js)
- Mobile Apps
- Desktop Applications
- Cloud Services
- Automation
- Game Development
- Machine Learning
- AI-powered Applications
- Internet of Things (IoT)

# Timeline

```text
1995  → JavaScript created by Brendan Eich
1995  → Netscape Navigator 2.0

2009  → ES5

2016+ → Annual ECMAScript Releases

```

# Interesting Facts

- JavaScript and Java are different languages.

# Summary

JavaScript began as a small scripting language for browsers but quickly became the foundation of modern web development. Through ECMAScript standardization and continuous improvements, it evolved into a powerful, general-purpose programming language used across multiple platforms.

Understanding its history provides valuable context for why JavaScript includes certain features and how it became the dominant language of the web.

# Key Takeaways

- JavaScript was created in **1995**.
- Brendan Eich developed JavaScript in **10 days**.
- It was first released in Netscape Navigator.
- JavaScript was originally named **Mocha**, then **LiveScript**.
- ECMAScript is the official JavaScript standard.
- ES6 (2015) introduced many modern language features.
- JavaScript now powers web, server, mobile, desktop, cloud, and IoT applications.
