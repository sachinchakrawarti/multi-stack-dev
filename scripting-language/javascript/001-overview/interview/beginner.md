## 1. What is JavaScript?

<div class="answer-box">

### 🎯 Answer:

**JavaScript** is a high-level, interpreted, dynamically typed programming language used to create interactive and dynamic web applications. It runs in web browsers and server-side environments like Node.js.

</div>

### 📝 Key Points:

- **High-level**: Abstracted from machine code
- **Interpreted**: Executed line by line
- **Dynamically typed**: Variables can hold any type
- **Runs in**: Browsers & Node.js

---

## 2. Who created JavaScript?

<div class="answer-box">

### 🎯 Answer:

JavaScript was created by **Brendan Eich** in **1995** while working at **Netscape Communications**.

</div>

### 📝 Fun Facts:

- Created in just **10 days**
- Originally called **Mocha**
- Later renamed to **LiveScript**
- Finally named **JavaScript**

---

## 3. How long did it take to create JavaScript?

<div class="answer-box">

### 🎯 Answer:

The first version of JavaScript was developed in approximately **10 days**.

</div>

### 📝 Timeline:

| Duration | Event |
|----------|-------|
| 10 days | Initial creation |
| 1995 | Released with Netscape Navigator |
| 1997 | Standardized as ECMAScript |

---

## 4. What was JavaScript originally called?

<div class="answer-box">

### 🎯 Answer:

JavaScript was initially named:

| Name | Stage |
|------|-------|
| **Mocha** | Original name |
| **LiveScript** | Second name |
| **JavaScript** | Final name |

</div>

---

## 5. Why was JavaScript created?

<div class="answer-box">

### 🎯 Answer:

JavaScript was created to make web pages **interactive** by allowing code to run directly inside web browsers.

</div>

### 📝 Use Cases:

- ✅ Form validation
- ✅ Button click events
- ✅ Animations
- ✅ Dynamic content updates
- ✅ Interactive maps
- ✅ Real-time updates

---

## 6. Is JavaScript the same as Java?

<div class="answer-box">

### 🎯 Answer:

**No.** Java and JavaScript are completely different programming languages.

</div>

### 📝 Comparison Table:

| Feature | Java | JavaScript |
|---------|------|------------|
| **Compilation** | Compiled | Interpreted/JIT |
| **Runtime** | JVM | Browser & Node.js |
| **Typing** | Static | Dynamic |
| **Creator** | James Gosling | Brendan Eich |
| **Year** | 1995 | 1995 |

---

<div class="answer-box">

JavaScript can run in multiple environments:

## 9. What is Node.js?

<div class="answer-box">

### 🎯 Answer:

**Node.js** is a JavaScript runtime that allows JavaScript to execute outside the browser. It is mainly used for backend development.

</div>

### 📝 Key Features:

- **Built on**: Chrome's V8 engine
- **Created by**: Ryan Dahl
- **Year**: 2009
- **Package Manager**: npm

---

## 10. What is ECMAScript?

<div class="answer-box">

### 🎯 Answer:

**ECMAScript (ES)** is the official standard specification that defines how JavaScript should work. JavaScript is an implementation of the ECMAScript standard.

</div>

### 📝 ECMAScript Versions:

| Version | Year | Key Features |
|---------|------|--------------|
| **ES1** | 1997 | First edition |
| **ES5** | 2009 | Strict mode, JSON |
| **ES6** | 2015 | Classes, Modules, Arrow functions |
| **ES2016+** | 2016+ | Annual updates |

---

## 11. What is a JavaScript Engine?

<div class="answer-box">

### 🎯 Answer:

A **JavaScript Engine** is software that reads, interprets, compiles, and executes JavaScript code.

</div>

### 📝 Popular JavaScript Engines:

| Engine | Browser/Environment |
|--------|-------------------|
| **V8** | Chrome, Node.js, Edge |
| **SpiderMonkey** | Firefox |
| **JavaScriptCore** | Safari |

---

## 12. What is the JavaScript Runtime?

<div class="answer-box">

### 🎯 Answer:

The **JavaScript Runtime** consists of multiple components that work together to execute JavaScript code.

</div>

### 📝 Components:

- **JavaScript Engine** - Executes code
- **Memory Heap** - Stores objects
- **Call Stack** - Manages function execution
- **Web/Node APIs** - Browser or Node features
- **Callback Queue** - Stores pending callbacks
- **Event Loop** - Moves callbacks to Call Stack

---

## 13. What is the Call Stack?

<div class="answer-box">

### 🎯 Answer:

The **Call Stack** is a data structure that manages function execution in JavaScript. Functions are added (pushed) when called and removed (popped) after execution.

</div>

### 💻 Example:

```javascript
function first() { console.log("First"); second(); }
function second() { console.log("Second"); third(); }
function third() { console.log("Third"); }
first();

// Output: First, Second, Third
