## Why Learn Common Mistakes?

Understanding common errors helps you:

- Debug faster
- Write better code
- Avoid unexpected behavior
- Improve interview performance

---

## Mistake 1: Using `var` Everywhere

❌

```javascript
var name = "Sachin";
```

✅

```javascript
const name = "Sachin";
```

Use `let` only when the value changes.

---

## Mistake 2: Accessing Variables Before Declaration

❌

```javascript
console.log(age);

let age = 20;
```

Output

```
ReferenceError
```

---

## Mistake 3: Reassigning a `const`

❌

```javascript
const PI = 3.14;

PI = 3.14159;
```

Output

```
TypeError
```

---

## Mistake 4: Redeclaring `let`

❌

```javascript
let city = "Delhi";

let city = "Mumbai";
```

Output

```
SyntaxError
```

---

## Mistake 5: Poor Variable Names

❌

```javascript
let a;
let b;
let x;
```

✅

```javascript
let studentAge;

let employeeSalary;

let productPrice;
```

---

## Mistake 6: Global Variables

❌

```javascript
let total = 0;
```

Used everywhere.

Better

```javascript
function calculate() {

    let total = 0;

}
```

Keep variables local whenever possible.

---

## Mistake 7: Forgetting Initialization

❌

```javascript
let username;

console.log(username);
```

Output

```
undefined
```

Better

```javascript
let username = "";
```

---

## Mistake 8: Confusing `null` and `undefined`

```javascript
let name;

console.log(name);
```

Output

```
undefined
```

```javascript
let city = null;

console.log(city);
```

Output

```
null
```

They represent different concepts.

---

## Mistake 9: Assuming Objects Are Immutable with `const`

```javascript
const student = {

    name: "Sachin"

};

student.name = "Rahul";
```

This is valid.

Only the variable reference is constant—not the object's contents.

---

## Mistake 10: Ignoring Scope

❌

```javascript
if (true) {

    let age = 25;

}

console.log(age);
```

Output

```
ReferenceError
```

Variables declared with `let` and `const` exist only within their block.

---

## Summary

Avoid these mistakes:

- Using `var`
- Accessing variables before declaration
- Reassigning `const`
- Redeclaring `let`
- Poor naming
- Large global scope
- Uninitialized variables
- Confusing `null` and `undefined`
- Misunderstanding object mutability
- Ignoring block scope
