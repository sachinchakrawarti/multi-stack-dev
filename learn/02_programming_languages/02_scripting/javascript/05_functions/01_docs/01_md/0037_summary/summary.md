# Key Concepts

Some of the most important concepts from this module include:

- Functions are reusable blocks of code.
- JavaScript treats functions as **first-class objects**.
- Functions can accept other functions as arguments.
- Functions can return other functions.
- Closures allow inner functions to access variables from outer scopes.
- Arrow Functions have a lexical `this`.
- Higher-Order Functions enable functional programming.
- Pure Functions improve maintainability.
- Side Effects should be minimized and isolated.
- Hoisting affects Function Declarations differently from Function Expressions.

# Best Practices Recap

Always remember to:

- Write small functions.
- Give functions meaningful names.
- Keep one responsibility per function.
- Prefer Pure Functions whenever possible.
- Avoid unnecessary Side Effects.
- Use default parameters.
- Avoid global variables.
- Keep functions reusable.
- Write readable code.
- Test functions independently.

---

# Common Mistakes Recap

Avoid these common mistakes:

- Forgetting `return`.
- Calling functions incorrectly.
- Misusing `this`.
- Overusing global variables.
- Writing very large functions.
- Ignoring default parameters.
- Mutating objects unnecessarily.
- Confusing parameters with arguments.
- Depending too much on hoisting.
- Mixing business logic with UI logic.

# Quick Revision Notes

| Topic | Key Point |
|--------|-----------|
| Function Declaration | Fully hoisted |
| Function Expression | Not fully hoisted |
| Arrow Function | Lexical `this` |
| Callback | Function passed to another function |
| Higher-Order Function | Accepts or returns functions |
| Closure | Remembers outer variables |
| Scope | Controls variable accessibility |
| Rest Parameter | Collects multiple arguments |
| Spread Operator | Expands iterable values |
| Pure Function | No Side Effects |
| Side Effect | Changes external state |
| IIFE | Executes immediately |
| Async Function | Uses `await` for asynchronous operations |
