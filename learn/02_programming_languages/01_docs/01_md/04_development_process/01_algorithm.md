# Algorithm

## Introduction

Before writing a program, programmers first think about **how to solve the problem**. Instead of writing code immediately, they create a clear sequence of steps that describes the solution. This sequence of steps is called an **algorithm**.

Algorithms are the foundation of programming and computer science. Every software application, from a simple calculator to a complex search engine, uses algorithms to solve problems efficiently.

---

# Definition

An **algorithm** is a finite, ordered sequence of well-defined instructions used to solve a specific problem or perform a particular task.

In simple words,

> An algorithm is a step-by-step procedure for solving a problem.

---

# Why Do We Need Algorithms?

Algorithms help programmers:

- Solve problems systematically
- Plan solutions before coding
- Reduce logical errors
- Improve code quality
- Optimize performance
- Make programs easier to understand and maintain

Without algorithms, programming becomes difficult and unorganized.

# Characteristics of a Good Algorithm

A good algorithm should have the following characteristics.

## 1. Input

It should accept zero or more inputs.

Example:

```
Two numbers
```

---

## 2. Output

It should produce at least one output.

Example:

```
Sum of two numbers
```

---

## 3. Definiteness

Each step should be clear and unambiguous.

Bad:

```
Add a large number.
```

Good:

```
Add 50 to the total.
```

---

## 4. Finiteness

The algorithm must end after a finite number of steps.

It should never run forever unless intentionally designed to do so.

---

## 5. Effectiveness

Every step should be simple, practical, and executable.

---

# Structure of an Algorithm

A basic algorithm generally consists of three parts.

```text
Start
   │
Input
   │
Process
   │
Output
   │
Stop
```

---

# Example Algorithm

## Problem

Find the sum of two numbers.

### Algorithm

1. Start.
2. Read the first number.
3. Read the second number.
4. Add both numbers.
5. Display the result.
6. Stop.

---

# Another Example

## Problem

Determine whether a person can vote.

### Algorithm

1. Start.
2. Read the person's age.
3. If age is greater than or equal to 18:
   - Display "Eligible to vote."
4. Otherwise:
   - Display "Not eligible to vote."
5. Stop.

# Representation of Algorithms

Algorithms can be represented in different ways.

## 1. Simple Steps

```
Step 1
Step 2
Step 3
```

---

## 2. Pseudocode

A language-independent representation of an algorithm.

Example:

```text
START

Read A
Read B

SUM = A + B

Print SUM

STOP
```

---

## 3. Flowchart

A graphical representation using symbols.

```text
Start
   │
Input
   │
Process
   │
Output
   │
Stop
```

# Advantages of Algorithms

- Easy to understand
- Independent of programming languages
- Helps detect logical errors
- Simplifies coding
- Makes debugging easier
- Encourages code optimization
- Improves communication among developers

---

# Disadvantages of Algorithms

- Time-consuming for very large problems
- Complex algorithms can become difficult to understand
- Does not show detailed implementation
- May require additional diagrams such as flowcharts

---

# Algorithm vs Program

| Algorithm | Program |
|-----------|---------|
| Step-by-step solution | Actual implementation in a programming language |
| Language independent | Language dependent |
| Easy to modify | Requires code changes |
| Focuses on logic | Focuses on implementation |

---

# Algorithm vs Flowchart

| Algorithm | Flowchart |
|-----------|-----------|
| Written in text | Graphical representation |
| Easier to write | Easier to visualize |
| Suitable for documentation | Suitable for explaining program flow |

# Best Practices

- Understand the problem before writing an algorithm.
- Keep the steps simple and clear.
- Avoid unnecessary steps.
- Break complex problems into smaller parts.
- Test the algorithm with different inputs.

---

# Common Mistakes

- Writing vague instructions.
- Missing important steps.
- Creating infinite loops.
- Ignoring edge cases.
- Jumping directly to coding without planning.

# Summary

- An algorithm is a step-by-step procedure for solving a problem.
- Algorithms are language independent and focus on logic rather than implementation.
- Every good algorithm should have input, output, definiteness, finiteness, and effectiveness.
- Algorithms can be represented using simple steps, pseudocode, or flowcharts.
- Writing a good algorithm before coding leads to better software design and fewer errors.
