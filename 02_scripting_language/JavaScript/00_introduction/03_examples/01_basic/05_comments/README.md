# 05 - JavaScript Comments

## 📖 Overview

This example demonstrates how to write **comments** in JavaScript.

Comments are ignored by the JavaScript engine and are used to explain code, improve readability, and make programs easier to maintain.

## 📌 Types of Comments

### 1. Single-Line Comment

Use `//` for comments that occupy a single line.

```javascript
// This is a single-line comment
console.log("Hello");
```

---

### 2. Multi-Line Comment

Use `/* ... */` for comments that span multiple lines.

```javascript
/*
This is a
multi-line comment.
*/

console.log("Hello");
```

## 💡 Best Practices

- Write comments that explain **why**, not just **what**.
- Keep comments short and meaningful.
- Update comments when code changes.
- Avoid obvious comments.

Good:

```javascript
// Validate user input before saving
```

Avoid:

```javascript
// Increment i by 1
i++;
```

---

## ❌ Common Mistakes

- Writing outdated comments.
- Commenting every line unnecessarily.
- Using comments instead of meaningful variable names.
- Leaving large blocks of commented-out code in production.
