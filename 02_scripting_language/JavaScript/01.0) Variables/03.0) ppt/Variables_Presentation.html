📦 What is a Variable?

A variable is a named container for storing data in memory.

📦 Storage Box Analogy

  • Label = Variable name (age)
  • Contents = Value (25)
  • Box = Memory address

⚡ Why Variables?

  • 🔹 Reusability — Use values multiple times
  • 🔹 Readability — Human-readable names
  • 🔹 Maintainability — Update in one place

🧠 Memory Architecture

📚 The Stack

  • ✅ Primitive values (number, string)
  • ✅ Variable identifiers
  • ✅ Function execution contexts
  • ⚡ Speed: Fast & organized

🗄️ The Heap

  • ✅ Objects, Arrays, Functions
  • ✅ References (pointers)
  • ✅ Large, unstructured space
  • 📏 Size: Dynamic & flexible
// Stack stores primitive values
let age = 25;   // ✅ Stored in stack
const user = { name: 'John' };   // Stored in heap (reference in stack)

🔄 Declaration vs. Initialization

📋 Declaration

Registering the variable in scope

let score;   // Declaration only
// Defaults to 'undefined'

🔹 Memory allocated but empty

⚡ Initialization

Assigning first value

let score = 100;   // Declaration + Initialization

🔹 Memory allocated and filled

💡 Key Insight: const requires both declaration and initialization in one step.

⚠️ var — The Legacy Keyword

Issues with var

  • ❌ Function-scoped, not block-scoped
  • ❌ Hoisted with undefined
  • ❌ Can be redeclared
  • ❌ Creates global scope pollution
var x = 10;
if (true) {
  var x = 20; // Same variable!
}
console.log(x); // Output: 20

🔸 Unexpected behavior — use let instead!

🚀 let — Modern & Mutable

Benefits of let

  • ✅ Block-scoped ({ })
  • ✅ No redeclaration allowed
  • ✅ Temporal Dead Zone (TDZ)
  • ✅ Mutable — can be reassigned
let count = 0;
if (true) {
  let count = 5; // New variable in block
  console.log(count); // 5
}
console.log(count); // 0

🔒 const — Immutable Reference

Rules of const

  • 🔒 Block-scoped like let
  • 🔒 Cannot be reassigned
  • 🔒 Must initialize during declaration
  • 🔒 Object properties can be modified
const PI = 3.14159;
// PI = 3.14; ❌ TypeError

const user = { name: 'Alice' };
user.name = 'Bob'; // ✅ Allowed
// user = {}; ❌ TypeError

🔍 Scope Explained

🌍

Global Scope

  • Accessible everywhere
  • Declared outside functions
  • Avoid pollution
📦

Function Scope

  • Inside functions
  • var is function-scoped
  • Accessible only within
🧱

Block Scope

  • Inside { }
  • let & const
  • Most restrictive
let global = 'I am global'; // Global Scope
function example() {
  var functional = 'Function Scope'; // var
  let blockScoped = 'Block Scope';   // let
}

🏗️ Hoisting — The Illusion

⬆️ What is Hoisting?

  • Declarations moved to top
  • JavaScript's compilation phase
  • var → undefined
  • let & const → TDZ
// Code before hoisting:
console.log(x); // undefined
var x = 5;

// After hoisting:
var x;
console.log(x); // undefined
x = 5;

⏳ Temporal Dead Zone (TDZ)

TDZ Explained

  • Time between entering scope and declaration
  • Variable exists but cannot be accessed
  • Affects let and const
  • Prevents accidental usage
// TDZ starts here for 'x'
{
  // TDZ for 'x' continues
  console.log(x); // ❌ ReferenceError
  let x = 10; // TDZ ends
  console.log(x); // ✅ 10
}

💡 Best Practices

✅ Do's

  • ✅ Use const by default
  • ✅ Use let when reassignment is needed
  • ✅ Use meaningful variable names
  • ✅ Declare variables at the top
  • ✅ Use camelCase naming convention

❌ Don'ts

  • ❌ Avoid using var
  • ❌ Don't use single-letter names
  • ❌ Avoid global variables
  • ❌ Don't use reserved keywords
  • ❌ Never redeclare let or const

🧪 Common Mistakes

🔴 Mistake #1

// Forgetting const initialization

Always initialize const variables.

🔴 Mistake #2

// Using var in loops

  setTimeout(() => console.log(i), 100);
} // Outputs: 3, 3, 3
